# 教练会话编排

## 意图路由

先把用户请求归类为以下模式之一：

- 只问通用原则
- 判断关系或当前局面
- 生成回复建议
- 复盘聊天表现
- 仅查询证据或上下文

## 决策树

1. 如果用户只是问通用原则，可以直接回答，但要说明这不是基于具体聊天样本
2. 如果用户是在问某个真实对象、某段真实互动，先检查是否已有足够证据
3. 证据不足时，先走 WeFlow 数据访问流程
4. 如果用户主要在问“怎么回”，且最近聊天已经足够，优先交给 `reply-strategy-coach`
5. 只有问题确实依赖长期模式时，才引入已有画像或触发更深入分析

## 输出模式

- 短答模式：简短判断 + 一两条下一步建议
- 完整分析模式：阶段、舒适度、主动性、边界感、风险点、证据
- 候选回复模式：优先走 `reply-strategy-coach`，输出快速读局面 + 几条自然回复选项

## 收敛原则

- 证据混杂时保持保守，不要硬给确定性结论
- 优先维护舒适感和体面，而不是强求推进
- 如果用户试图从弱信号里榨出确定答案，要主动收窄结论范围
