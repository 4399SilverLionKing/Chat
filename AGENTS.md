# 项目级指令

## 项目 Skills

本仓库提供项目级 skills，目录位于 `.agents/skills/`。

当用户提出以下类型的请求时，应优先匹配对应 skill：

- 恋爱或暧昧聊天分析
- 关系阶段判断
- 基于聊天证据的回复建议
- 聊天复盘
- 基于 WeFlow 的联系人或消息查询

## WeFlow 数据访问

当需要外部聊天数据时：

- 优先调用仓库内 CLI 命令，不要临时手写 HTTP 请求
- 先确认联系人，再读取聊天上下文
- 如果联系人不明确，应要求提供 `wxid` 或 `wechat_id`，不要猜测

## 分析约束

- 有聊天证据时，结论必须建立在证据之上
- 明确区分高置信判断和弱推测
- 证据不足时要直接说明，不要硬下结论
- 不要把普通礼貌或友好强行解读成明确好感

## 建议风格

- 建议要自然、具体、低压力
- 尊重边界感和相处节奏
- 不提供操控、施压、查岗、PUA 风格建议
- 优先帮助用户看清局面，而不是强推关系升级
