# 项目文档入口与上手教程重构设计

## 1. 背景

当前根目录 `README.md` 同时承担了项目介绍、命令清单、配置约定和 Codex 使用提示。对于第一次进入仓库的用户来说，首页信息偏散；对于已经准备开始使用的人来说，缺少一篇从安装到实际操作的连续教程。

用户要求将根目录 `README` 收敛为项目介绍页，把详细使用教程放到 `docs` 目录下，同时教程要面向仓库使用者，而不是只面向 Codex 使用者。

## 2. 目标

- 让 `README.md` 成为清晰的项目入口页
- 在 `docs/` 下提供一篇完整、连续的上手教程
- 明确区分“项目介绍”和“详细操作说明”的边界
- 保留本项目对 Codex / 项目级 skills 的说明，但不喧宾夺主

## 3. 非目标

- 不改动 CLI 功能和命令行为
- 不拆成多篇教程文档
- 不新增新的配置字段或命令别名
- 不把 `README` 扩写成完整命令手册

## 4. 文档信息架构

### 4.1 README 的职责

`README.md` 只保留：

- 项目是什么
- 适合谁使用
- 核心能力概览
- 仓库结构概览
- 最小化快速开始
- 指向完整教程的入口链接

### 4.2 docs 教程的职责

在 `docs/getting-started.md` 中提供一篇完整教程，覆盖：

- 项目简介与适用场景
- 环境准备与依赖安装
- 配置文件说明
- CLI 快速上手
- 常用命令详解
- Codex 与项目级 skills 的协作方式
- 推荐工作流
- 常见问题与排查线索

## 5. 内容边界原则

- `README` 只放少量关键命令示例，不逐项展开参数解释
- 命令参数、配置含义、输出目录和推荐流程统一收敛到 `docs/getting-started.md`
- 涉及 Codex 时，说明其是可选但推荐的协作方式，不把教程写成只对 Codex 有意义的内部文档

## 6. 结果预期

调整后，新用户会有两条清晰路径：

1. 先看 `README`，快速理解项目定位与能力范围
2. 再进入 `docs/getting-started.md`，按顺序完成安装、配置和实际使用
